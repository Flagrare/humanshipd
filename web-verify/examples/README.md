# Example credentials

A ready-made pair for trying the validator without writing anything yourself:

- `credential.c2pa` — a signed Human Authored credential.
- `document.txt` — the document it's bound to.

The validator's **Load a demo credential** button reads these directly; you can
also download both and try the normal upload flow. They describe a short,
mostly-typed session with one pasted block and a mid-document correction, so the
report shows "typed, with some pastes" and the fingerprint shows a paste jump and
a revisit dip.

These are generated by `cargo run --example issue_credential -- web-verify/examples`
and signed with c2pa's ephemeral (self-signed) signer — fine for a demo, untrusted
for anything real.
